Write Your AI Bot

Your bot is a Python file. The platform calls your code once per move, passing the current board position and expecting a legal move back. Implement an EndpointHandler class with a __call__(self, data) method.

♟ Chess Bot

Return a move in UCI notation (e.g. e2e4).

# handler.py
import chess, random
class EndpointHandler:
def __init__(self, path=""): pass
def __call__(self, data: dict) -> dict:
"""data = {"inputs": {"fen": "...", "player": "w", "game_type": "chess"}}"""
fen = data["inputs"]["fen"]
board = chess.Board(fen)
move = random.choice(list(board.legal_moves))
return {"move": move.uci()}

Create Your Hugging Face Repositories

You need two public repositories on Hugging Face — one for your model code, one for your data. Both must be public.

📦 Repo 1 — Model Repository (HF Model repo)

Must include a config_model.json at the root listing all your Python files and the entry-point module.

Required layout:
your-username/my-chess-bot/
├── config_model.json <-- required
├── handler.py <-- contains EndpointHandler
├── my_model.py <-- any helper modules
├── AGL_VERIFY.txt <-- added in step 3
└── requirements.txt
config_model.json
{
"files": ["handler.py", "my_model.py"],
"main": "handler"
}

The main key is the module name (no .py) that contains your EndpointHandler class.

🗂️ Repo 2 — Data Repository (HF Dataset repo)

A Hugging Face dataset repo containing runtime data files (zone DBs, opening books, weights). Must include config_data.json. For Chess with no extra data, use a minimal config.

Required layout:
your-username/my-breakthrough-data/
├── config_data.json <-- required
└── breakthrough_zone_db.npz <-- your data file(s)
config_data.json (Breakthrough)
{
"files": ["breakthrough_zone_db.npz"],
"zone_db": "breakthrough_zone_db.npz"
}
config_data.json (Chess / no data files)
{"files": []}

Prove Ownership & SHA Pinning

Before your model can play, you must prove you own the repository. The platform also pins the commit SHA to detect any changes during tournaments.

Ownership Verification (one-time per repo)

  1. 1 Register your repo on the AI Models page. The platform generates a unique 32-character verification code.
  2. 2 Create a file named AGL_VERIFY.txt at the root of your HF model repo with the exact code shown — no extra characters or whitespace.
  3. 3 Click "Verify Ownership". The platform downloads your AGL_VERIFY.txt and confirms the code matches.

SHA Pinning (ongoing)

  • ▸At tournament registration — the exact commit SHA of your repo is recorded.
  • ▸Before each tournament round — the SHA is re-checked. If your repo changed since registration, you are automatically disqualified.
⚠️ Do not push changes to your model repo after joining a tournament. SHA mismatch = automatic disqualification.
✅ No Hugging Face token required. All checks use your public repo — no credentials needed from you.

Register Your Model in the Platform

Go to Profile → AI Models and fill in the repo IDs for each game you want to play. No token needed.

♟ Chess
  • • Model Repo ID — username/repo-name
  • • Data Repo ID — username/data-repo (optional)
🔴 Breakthrough
  • • Model Repo ID — username/bt-repo
  • • Data Repo ID — username/bt-data (recommended — holds zone DB)
⚠️ After registering, you need 30 rated games with that model before you can enter tournaments. Updating your repo resets this counter.

How a move is generated
Board FEN sent → EndpointHandler.__call__ invoked → Move validated → Game state updated

Each call has a strict time limit. Exceeding it forfeits the move. Keep inference fast.

❓ Frequently Asked Questions

Can I use any ML framework (PyTorch, TensorFlow, scikit-learn…)?

Yes. List dependencies in requirements.txt. The platform installs them in a sandboxed environment before running your handler.

Do I need a Hugging Face token?

No. Your repos must be public. No credentials are required from you.

What happens if I update my model repo?

Outside tournaments you can update freely, then re-verify. Inside a tournament, updating after registration triggers an automatic disqualification (SHA mismatch).

Can I submit different models for Chess and Breakthrough?

Yes. Each game type is registered independently with its own model and data repos.

What move format does Breakthrough use?

Coordinate strings: column + row, e.g. e2e3 (from e2 to e3 on an 8×8 board).

What is the zone DB in config_data.json?

A precomputed lookup table (e.g. .npz) for fast Breakthrough move evaluation. The zone_db key names which file in your data repo is the zone DB.

Why do I need 30 rated games before tournaments?

It ensures your Elo is meaningful and your model is stable before entering a structured competition.
